# Garmin Agent 使用场景分析

**日期**：2026-05-18
**方法**：用户描述真实使用场景 → 分析每个场景的数据需求、组合方式、呈现方式

---

## 时间轴：用户什么时候需要这个 Agent

```
跑完步（最频繁）
  │
  ├── 单次评估：今天跑得怎么样？
  ├── 同类对比：在我的轻松跑/节奏跑里排第几？
  ├── 原因分析：为什么好/不好？
  └── 下一步建议：接下来该怎么跑？

周/月周期
  │
  ├── 周回顾：这周训练量/强度怎么样？
  └── 月回顾：这个月能力有没有提升？

备赛/换季
  │
  ├── 备赛规划：前面跑得怎样？目标定多少？怎么准备？
  └── 基础期评估：能力提升到什么程度？下阶段怎么练？
```

---

## 场景一：跑完步后的评估（最频繁）

### 用户问题示例
- "我今天跑了个轻松跑，跑得怎么样？"
- "今天这个节奏跑和以前的节奏跑比呢？"
- "今天间歇跑的质量怎么样？"

### 我需要知道的
1. 今天的活动数据（配速、心率、步频、步幅、垂振、圈数据）
2. 活动分类（轻松跑？节奏跑？间歇？长距离？）
3. 历史同类活动数据（同分类的历史活动）

### 我需要组合的
- 今天 vs 最近3-5次同类活动的关键指标对比
- 同类活动的趋势（配速是越来越快还是稳定？心率是越来越低还是稳定？）

### 我需要呈现的
1. 本次训练的基本数据
2. 在同类训练中的位置（"这是你最近第X次轻松跑，配速排第Y"）
3. 与同类训练的对比表
4. 简短评价（训练效果、恢复状态、需要注意的点）

### 用户验证问题
- 这个呈现方式够不够？
- 需要更详细的圈级对比吗？
- 需要跟去年同期对比吗？

## 场景二：节奏跑/间歇跑的深度对比

### 用户问题示例
- "今天这个节奏跑和去年这个时候比呢？"
- "这个间歇跑为什么跑得好？感觉很好"
- "今天的节奏跑不好，帮我分析一下原因"

### 我需要知道的
1. 今天的圈级数据（每个圈的配速、心率、步频、步幅）
2. 历史同类活动的圈级数据
3. 去年同期（±2周）的同类活动数据
4. 可能的环境因素（天气、场地、训练负荷）

### 我需要组合的
- 今天的圈数据 vs 历史同类圈数据的逐圈对比
- 配速-心率-步频的关联分析
- "好"的模式识别（哪些指标组合时感觉好）
- "差"的原因追溯（哪些指标异常）

### 我需要呈现的
1. 逐圈对比表（今天 vs 历史最优 vs 历史平均）
2. 关键指标的变化趋势
3. 原因分析（如果跑得好：哪些指标组合说明状态好；如果跑得差：哪个环节出了问题）

### 用户验证问题
- 这个深度够不够？
- 原因分析需要多精确？
- "感觉好"这种主观信息怎么纳入？

## 场景三：周/月回顾

### 用户问题示例
- "这周跑得怎么样？"
- "这个月的训练量和上个月比呢？"
- "这个月有没有进步？"

### 我需要知道的
1. 本周/本月所有活动数据
2. 上周/上月所有活动数据
3. 每个活动的分类

### 我需要组合的
- 训练量统计（总距离、总时间、活动次数）
- 强度分布（轻松跑/节奏跑/间歇各占多少）
- 同类活动的月度趋势
- 关键指标的变化（如：轻松跑的平均配速/心率趋势）

### 我需要呈现的
1. 训练量/强度分布饼图或表格
2. 与上个周期的对比
3. 同类活动的趋势指标
4. 整体评价

### 用户验证问题
- 需要精确到什么程度？
- 需要可视化还是纯文字够了？
- 需要和训练计划对比吗？

## 场景四：备赛规划

### 用户问题示例
- "我要备赛半马，前面跑得怎么样？目标定多少？"
- "备赛的话，接下来怎么安排？"

### 我需要知道的
1. 最近3个月的训练数据
2. 最近的比赛成绩（如果有）
3. 用户的目标或期望

### 我需要组合的
- 当前能力评估（基于最近的训练数据）
- 与目标的差距分析
- 历史比赛数据与训练数据的关联

### 我需要呈现的
1. 当前能力水平评估
2. 建议目标（基于数据推算）
3. 备赛期的训练框架建议

### 用户验证问题
- 这个场景需要多强的"建议"能力？
- 需要接入训练计划模板吗？

## 场景五：基础期能力评估

### 用户问题示例
- "进入夏天了，前面跑得怎么样？能力提升了多少？"
- "接下来的基础训练期应该怎么安排？"

### 我需要知道的
1. 最近2-3个月的训练数据
2. 去年同期的训练数据（如果有的话）
3. 训练分类

### 我需要组合的
- 能力指标的趋势（轻松跑配速、心率恢复速度等）
- 同期对比（今年 vs 去年）
- 训练量/强度的变化

### 我需要呈现的
1. 能力指标的变化趋势
2. 与去年同期的对比
3. 基础期训练建议

### 用户验证问题
- 能力提升用什么指标衡量？
- 需要哪些具体指标的趋势？

## 初步架构推演

根据以上场景，数据需求可以分为几类：

### 数据获取层（Layer 1 - API 映射）
所有场景都需要的基础数据：
- 活动列表（按日期、按类型）
- 活动详情（配速、心率、距离、时间）
- 圈数据（逐圈指标）
- 分类信息

### 数据组合层（Layer 2 - Join/Chain）
场景需要的组合操作：
- **同类活动聚合**：按分类筛选 + 时间排序 + 指标提取
- **周期统计**：按周/月聚合 + 训练量/强度计算
- **同比对比**：今年 vs 去年同期的同类活动对比
- **趋势计算**：同类活动的关键指标随时间变化

### 分析计算层（Layer 3 - 预定义脚本）
需要程序化计算的：
- 配速-心率回归分析
- 能力指标趋势计算
- 训练负荷计算
- 目标推算（基于历史数据预测完赛时间）

### 呈现层（LLM 负责）
LLM 最终要做的：
- 基于结构化数据写评价
- 用自然语言解释趋势
- 给出训练建议
